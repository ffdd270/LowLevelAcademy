컴퓨터에 저장된 모든 정보는 숫자로 인코딩 됩니다.
지금 읽고 있는 문장은 일련의 숫자들일 뿐 입니다. "hello"라는 단어는, 알파벳마다 각 글자에 해당 하는 5 개의 숫자들로 표시됩니다. <Dec>104</Dec>, <Dec>101</Dec>, <Dec>108</Dec>, <Dec>108</Dec>, <Dec>111</Dec>. 이렇게요.

우리는 UTF-8이라는 표준 인코딩 체계를 사용 하는데, 이는 기호, 모든 종류의 언어의 문자 및 이모티콘을 포함하여 많은 문자를 나타낼 수 있습니다. 다른 인코딩 시스템이 있지만, 지금은 UTF-8이 가장 일반적으로 사용됩니다.

아래 예제를 통해, 숫자가 어떻게 글자로 바뀌는 지 확인해보세요!

## 한국어는 어떻게 되나요? ( 한국어 역주. )
그런데 알파벳 하나에 숫자 하나에 해당하는 건 알겠지만. 그러면 한글은 어떻게 될까요? 큰 수 하나로 표현될 수도 있겠네요. 하지만 UTF-8은 8비트 단위로 단어를 표현하기에 UTF-8이라는 이름을 가지게 되었습니다. 8비트는 0xff, 즉 255 이상은 표기할 수 없으니 큰 수로는 한글을 표기할 수 없습니다.
그래서 UTF-8은 한글이나 한자, 읿본어처럼 비영어권 언어들을 표기할 때는 8비트를 여러개 사용하여 하나의 글자를 표기합니다. 그리고 여러개가 몇개인지 구분하는 방법은 첫 8비트에 적어 놨습니다. 특정 범위는 8비트 하나. 다른 범위는 8비트 두개.. 이런 식으로요.

## 10진수와 2진수

일상 생활에서 우리는 숫자 10을 기반인 _10진수_ 를 사용합니다. 수십, 수백, 수천, 기타등등.. 으로 샐 수 있습니다.
이걸로 우리는 나무 하나, 사과 두 개, 손가락 열 개등 물리적인 것들을 샐 수 있습니다.
하지만 다른 표기법과, 다른 수치 시스템을 이용해서 동일한 수량을 나타낼 수도 있습니다.
예를 들면, 때때로 사용하는 로마 숫자처럼: I, II, III, IV, X 같이요.
그렇지만 이 숫자는 많은 수들을 다루기에는 부적합한 것처럼, 십진수도 컴퓨터와 데이터를 다룰 때 항상 실용적이지는 않습니다.

숫자 <Dec>10</Dec>을 지수화하는 방법에 주목하십시오. 이것은 _number base_ 또는 _radix_ 이며, 0부터 시작하여 계수에 사용할 수있는 자릿수로 볼 수 있습니다.
예를 들어, <Dec>10</Dec>진법은 <Dec>0</Dec>에서 <Dec>9</Dec>까지 10 자리고, <Dec>8</Dec>진법은 8 자리입니다. <Oct>0</Oct>에서 <Oct> 7</Oct>까지요,  <Dec>2</Dec>진법은 <Bin> 0</Bin> 과 <Bin>1</Bin>의 두 숫자만 있습니다. 지수는 숫자의 자리로도 볼 수 있습니다.
<Dec>500</Dec>과 같은 숫자를 표기할 때는, <Dec>5</Dec>, <Dec>0</Dec>, <Dec>0</Dec>의 3자리 숫자입니다. <br/><Dec>5</Dec>는 오른쪽에서 _3<sup>번째</sup>_ 자리입니다. 숫자의 행을 0 기반 목록 또는 배열로 생각할 수 있습니다.
오른쪽에서 세어 보면 <Dec>5</Dec>는 색인 <Dec>2</Dec>에 있습니다. 따라서 <Dec>10</Dec>에 <Dec>2</Dec>의 거듭 제곱을 취하고 숫자 <Dec>5</Dec>를 곱합니다. 즉, <Dec>10<sup> 2</sup> </Dec> × <Dec>5</Dec> = <Dec>500</Dec>.

Notice how we exponentiate the number <Dec>10</Dec>: it's our _number base_, or _radix_, which can be seen as the number of digits that we can use for counting, starting with zero.
For example, base <Dec>10</Dec> has 10 digits from <Dec>0</Dec> to <Dec>9</Dec>, base <Dec>8</Dec> has 8 digits from <Oct>0</Oct> to <Oct>7</Oct>, and base <Dec>2</Dec> has only two digits: <Bin>0</Bin> and <Bin>1</Bin>. An exponent can also be seen as a digit's place.
If we take a number like <Dec>500</Dec>, it has 3 digits: <Dec>5</Dec>, <Dec>0</Dec>, and <Dec>0</Dec>.<br/><Dec>5</Dec> is the _3<sup>rd</sup>_ digit from the right. We can think of that row of numbers as a zero based list or array. Counting from the right, the <Dec>5</Dec> sits on index <Dec>2</Dec>. So we take <Dec>10</Dec> to the power of <Dec>2</Dec> and multiply by the digit <Dec>5</Dec>, i.e.: <Dec>10<sup>2</sup></Dec> × <Dec>5</Dec> = <Dec>500</Dec>.

숫자 밑을 변경하고 지수 합산의 동일한 원칙을 따를 때 재미 있는 일이 발생합니다.

## Hexadecimal

하지만 10 자리가 넘는 숫자 체계는 어떨까요? 예를 들어 _hexadecimal_, 그러니까 16 진법 숫자와 같이요.
숫자가 부족할 때는 문자를 대신 사용할 수 있습니다. 예를 들어 16 진수 <Hex> A </Hex>는 10 진수 <Dec> 10 </Dec>과 같습니다.
숫자 <Hex>B</Hex>는 <Dec>11</Dec>과 똑같고. 이 숫자는 <Hex>F</Hex>까지 올라갑니다.

But what about the number systems where we have more than 10 digits? Like, for example, _hexadecimal_, or the number base 16.
When we run out of digits, we can start using letters instead: for example, the hexadecimal digit <Hex>A</Hex> is equal to decimal <Dec>10</Dec>, and
the digit <Hex>B</Hex> is the same thing as <Dec>11</Dec>, and it goes up to <Hex>F</Hex>.

<HexPlayground />

16진수 시스템은 계산할 때 깔끔한 특징을 가지고 있어서, 우리가 사용하고 있습니다. 단일 _byte_ (<Dec>0</Dec>과 <Dec>255</Dec>, 아니면 2<sup>8</sup>) 는 단지 2개의 16진수 숫자로 나타낼 수 있습니다.
이를 통해서 우리는 데이터를 16진수들의 나열함으로써 깔끔한 그리드나, _헥스 뷰어_ 같은 걸 만들 수 있었습니다.

해당 텍스트 인코딩 예제로 돌아가 보겠습니다. 이번에는 16 진수 인코딩을 사용하겠습니다.

We use the hexadecimal number system in computing because it has a neat property: we can represent a single _byte_ (a number between <Dec>0</Dec>
and <Dec>255</Dec>, or 2<sup>8</sup>) with just 2 hexadecimal digits. This allows us to build neat grids, or _hex views_,
to represent data as sequences of hexadecimal numbers.

## Takeaways

컴퓨터는 본질적으로 디지털 기계이기 때문에 1과 0 만 처리합니다. 텍스트 인코딩 및 기본 변경을 통해
컴퓨터는 인간이 매일 사용하는 문자와 숫자를 처리 할 수 있습니다. 이제 몇 가지 기본 데이터가 인코딩되는 방법을 알았으니
"온라인"을 통해 네트워킹 및 저수준 프로그래밍에 대해 자세히 알아볼 수 있습니다.

By their very nature as digital machines, computers only deal with ones and zeros. Through text encoding and base changes,
computers can deal with the letters and numbers that humans use every day. Now that you know how some basic data is encoded
